Useful Command-line functions... that I always forget

Last updated on Jan 19, 2023 2 min read linux, cloud, cli

From linux

But just remember – easy to run in windows (e.g. have an ubuntu terminal installed)

find

trying to find where those files are?

find <dir> -name "*pattern*"  ## use wildcards *

Find files of a certain name/pattern/type – with the text pattern you are looking for. In the example, I search for Rmd files that call ggplot. See this StackExchange post for more details

grep --include=\*.{tyoe} -rnw '/path/to/somewhere/' -e "pattern"

grep --include=\*.Rmd -rnw . -e "ggplot"

rsync

I often share Rmarkdown reports via a website or cloud folder, but managing the changes and ensuring the website is up-to-date can be a bit of a pain. This is where rsync is awesome with one command, as long as I can remember it, I can push all the changes to the web.

The syntax:

rsync ^
    -av            # a to sync all, v for verbose mode
    --delete       # will remove files that exist in the destination but not the source 
    <source>
    <destination>

Here is an example for updating this website. This will cause a ssh tunnel to be established and the sync will be placed in the remote folder on the far side of the :

rsync -av ./public colin@chisholm.forestecosystems.ca:/home/colin/www/

Tip

For commands that are run numerous times – put them in a file and run the file. In windows end the file with .bat. For linux make it executable chmod 755 filename.
